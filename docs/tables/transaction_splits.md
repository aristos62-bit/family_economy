# 📄 transaction_splits collection (Firestore)

Ο πίνακας `transaction_splits` περιέχει τις αναλυτικές κατανομές μίας "split" συναλλαγής.  
Χρησιμοποιείται όταν μια ενιαία συναλλαγή περιλαμβάνει επιμέρους ποσά σε διαφορετικές κατηγορίες.

---

## 🧱 Πεδία

| Πεδίο | Τύπος | Περιγραφή |
|-------|-------|-----------|
| `uuid` | `string` | Μοναδικό ID για κάθε split |
| `parent_transaction_id` | `string` | ID της κύριας συναλλαγής |
| `category_id` | `string` | Κατηγορία για το μέρος αυτό της συναλλαγής |
| `subcategory_id` | `string \| null` | Υποκατηγορία (αν υπάρχει) |
| `amount` | `number` | Ποσό (π.χ. `-35.50`) |
| `notes` | `string \| null` | Προαιρετική περιγραφή |

---

## 🧠 Παρατηρήσεις

- Ο πίνακας είναι απαραίτητος **μόνο όταν `is_split = true` στην κύρια συναλλαγή**
- Το άθροισμα των `amount` σε όλα τα splits **πρέπει να ισούται με** το ποσό της κύριας συναλλαγής
- Δεν χρησιμοποιούνται timestamps ή συγχρονιστικά πεδία γιατί τα split records **συνδέονται άμεσα** με τη `transaction`

---

## 🔁 Συγχρονισμός

- Δεν χρειάζεται `user_id` ή `deleted`
- Αν η συναλλαγή συγχρονίζεται, τα splits έρχονται μαζί (ή επαναδημιουργούνται)

---
